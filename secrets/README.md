# Secrets

This directory documents what secrets the pipelines need. **Do not commit real keys here.** The `.gitignore` excludes anything ending in `.token` or `.key`.

The pipelines read from `~/.config/scan-pipeline/`, NOT from this repo.

## Files

| Path | Contents | Source |
|---|---|---|
| `~/.config/scan-pipeline/asana.token` | Asana Personal Access Token, one line, no quotes | Asana → Profile → Settings → Apps → Developer Apps → Manage Developer Apps |
| `~/.config/scan-pipeline/anthropic.key` | Anthropic API key starting with `sk-ant-api03-`, one line, no quotes | console.anthropic.com → API Keys |
| `~/.ssh/id_ed25519_n8n_backup` (+ `.pub`) | Dedicated SSH keypair for the backup pull. Generated by `backup-miniblog/install.sh` if missing. | Auto-generated |

## Perms

All files must be `0600`:

```bash
chmod 600 ~/.config/scan-pipeline/asana.token ~/.config/scan-pipeline/anthropic.key
chmod 600 ~/.ssh/id_ed25519_n8n_backup
```

## Paste cleanly

The biggest footgun: pasting a key with extra characters (quotes, redirect, trailing text).

**Right way:**
```bash
cat > ~/.config/scan-pipeline/anthropic.key   # paste key, then press Enter, then Ctrl-D
chmod 600 ~/.config/scan-pipeline/anthropic.key
```

**Wrong way (eats the redirect into the file):**
```bash
echo 'sk-ant-api03-... > ~/.config/scan-pipeline/anthropic.key' > ~/.config/scan-pipeline/anthropic.key
#         missing closing quote here ^
```

**Verify before relying on it:**
```bash
# Anthropic key should be exactly 108 chars (sometimes + trailing newline = 109)
wc -c < ~/.config/scan-pipeline/anthropic.key

# Asana token format: 2/<workspace_gid>/<random>
wc -c < ~/.config/scan-pipeline/asana.token

# Test Anthropic
KEY=$(cat ~/.config/scan-pipeline/anthropic.key)
curl -sS https://api.anthropic.com/v1/messages \
  -H "x-api-key: $KEY" -H "anthropic-version: 2023-06-01" -H "content-type: application/json" \
  -d '{"model":"claude-haiku-4-5-20251001","max_tokens":20,"messages":[{"role":"user","content":"PONG"}]}'
unset KEY

# Test Asana
TOKEN=$(cat ~/.config/scan-pipeline/asana.token)
curl -sS "https://app.asana.com/api/1.0/users/me" -H "Authorization: Bearer $TOKEN" | head -c 200
unset TOKEN
```

## Rotation

If you rotate either key, just replace the file contents. No restart needed — the script re-reads on every run.

## How the original Asana token got here

Originally minted in Asana → Developer Apps. Stored encrypted in the n8n credentials store. Migrated to this file when the scan pipeline moved off n8n on 2026-05-16. See `OPS.md` in this repo for the retrieval command if you need to re-extract from n8n.
